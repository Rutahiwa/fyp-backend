## <a id="phase-5"></a> 🔁 PHASE 5 — Password Reset (OTP Flow)

### Goal
Implement the full OTP-based password reset flow.

### Steps

#### 5.1 — POST `/api/auth/generate-otp` (`app/api/auth/generate-otp/route.ts`)
```
Request Body: { email: string }
Auth: None (public endpoint — RATE LIMITED)

Logic:
1. Validate input (generateOtpSchema)
2. Rate limit: 3 requests per 15 minutes per email → 429 if exceeded
3. Find user by email → return generic success even if not found (prevent enumeration)
4. Generate 6-digit OTP (crypto.randomInt)
5. Invalidate any existing unused OTPs for this user
6. Store OTP in otp table with expiresAt = now + OTP_EXPIRY_MINUTES
7. Send OTP email via Resend
8. Log audit action: "GENERATE_OTP"
9. Return generic success → 200 OK
```

#### 5.2 — POST `/api/auth/verify-otp` (`app/api/auth/verify-otp/route.ts`)
```
Request Body: { email: string, otpCode: string }
Auth: None (public endpoint — RATE LIMITED)

Logic:
1. Validate input (verifyOtpSchema)
2. Rate limit: 5 attempts per 15 minutes per email → 429 if exceeded
3. Find user by email
4. Find latest OTP for user where:
   - otpCode matches
   - expiresAt > now
   - usedAt is null
5. If not found → 400 "Invalid or expired OTP"
6. Mark OTP: usedAt = now()
7. Generate a short-lived reset token (JWT, 10 min expiry, purpose: "password-reset")
8. Log audit action: "VERIFY_OTP"
9. Return: { resetToken } → 200 OK
```

#### 5.3 — POST `/api/auth/reset-password` (`app/api/auth/reset-password/route.ts`)
```
Request Body: { resetToken: string, newPassword: string }
Auth: None (uses resetToken for authorization)

Logic:
1. Validate input (resetPasswordSchema)
2. Verify resetToken (must be valid JWT with purpose: "password-reset")
3. Fetch user from token payload
4. Hash newPassword
5. Update password in DB
6. Delete all OTP records for this user (cleanup)
7. Log audit action: "RESET_PASSWORD"
8. Return success → 200 OK
```

### Security Considerations
- OTP is 6 digits → must rate limit verification attempts
- Reset token is short-lived (10 min) → mitigates replay attacks
- Generic responses on generate-otp → prevents email enumeration
- Invalidate old OTPs when generating new ones → prevents OTP confusion

### Deliverables
- [x] OTP generation with email delivery
- [x] OTP verification with reset token issuance
- [x] Password reset with token validation
- [x] Rate limiting on all public OTP endpoints
